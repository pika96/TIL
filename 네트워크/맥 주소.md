# 맥 주소

## 맥 주소란?

> MAC 주소는 네트워크 세그먼트의 데이터 링크 계층에서 통신을 위한 네트워크 인터페이스에 할당된 고유 식별자이다.  
> MAC 주소는 이더넷과 와이파이를 포함한 대부분의 IEEE 802 네트워크 기술에 네트워크 주소로 사용된다.  
> 물리 주소나 하드웨어 주소 라고도 불린다.

## 맥 주소가 필요한 이유

> IP 주소는 시작점의 위치와 끝점에 해당하는 주소이다.  
> MAC 주소는 바로 옆에 나와 물리적으로 연결되어있는 노드와 통신할 때 사용되는 주소이다.  

네트워크를 공부하다보면 데이터를 송수신 하기 위해서 IP주소와 mac 주소 둘 다 필요하다고 한다. 목적지인 ip 주소만 있으면 되는 것이 아닌가? 라고 생각할 수도 있다.  
맥 주소는 모든 하드웨어(컴퓨터, 라우터, 스위치 등)에 붙어 있다.  
데이터를 보낼 때는 A에서 B까지 한번에 가는 것이 아닌 중간 중간에 연결되어 있다. 이 때 나와 이웃을 연결해주는 프로토콜이 데이터 링크 계층 프로토콜이고 이 때 필요한 주소가 맥 주소이다. 따라서 A가 B한테 데이터를 전송할 때 전송하는 데이터의 목적지 IP 주소뿐만 아니라 다음 맥 어드레스가 지정되어야 전송할 수 있다.

하지만 우리는 자신의 맥주소만 알지 상대방의 맥 주소는 모른다!  
이 때 사용되는 것이 ARP 프로토콜이다. ARP 프로토콜은 IP 주소로 부터 맥주소를 알아오는 기능을 한다.  

> ARP 프로토콜  
> ARP 프로토콜은 IP 주소로부터 다음 맥 주소를 알아올 수 있는 프로토콜이다.  
> - 송신자는 목적지 맥 주소가 필요하므로, 해당 맥 주소 요청을 위한 ARP 요청 패킷을 브로드캐스트로 전송한다. (브로드 캐스트 전송을 하는 이유는 목적지의 맥 주소를 모르므로 모두에게 요청)  
> - 모든 호스트와 라우터는 송신자가 보낸 ARP 요청 패킷을 수신한다.  
> - 해당되는 수신자만 목적지로 가는 길에 존재하는 MAC 주소를 유니캐스트로 응답한다.

> 맥 주소 테이블  
> 맥 주소 테이블은 캐싱이라고 생각하면 된다!  
> 맥 주소를 받아오게 되면 테이블에 저장하여 같은 요청이 왔을 때 ARP(브로드캐스트)를 보내지 않고 바로 보내준다!  
> 디폴트로 300초간 저장된다.

그렇다면 IP 주소가 필요가 없는게 아닐까?  
바로 옆에 있다면 맥주소를 날려 확인하면 되지만, 도착지가 머나먼 곳에 있다면 어디로 가야할지 갈피를 못잡을 것이다. 따라서 IP 주소라는 방향이 필요하다.  
IP 주소는 숲, 맥 주소는 옆 나무 라고 하면 이해가 될까..? ~~맞나..~~
